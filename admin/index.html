✅ 연결 상태
1

Apps Script 웹앱 URL 연결

tracker.gs를 배포한 뒤 나온 웹앱 URL을 입력하세요. config.js의 TRACKER_URL과 동일한 값이어야 합니다.

2

시트 구조 (v2)

이 버전은 3개의 원본 시트를 사용하고, 나머지 모든 통계는 대시보드 로드 시 그때그때 집계합니다.

sessions   — 세션 단위 (유입경로/기기/국가/체류시간)
pageviews  — 페이지·게시글·액티비티 열람 단위 (경로/체류시간/스크롤/이탈유형)
events     — 광고 노출·클릭, 어필리에이트 클릭, 툴 사용, 검색, 에러 등
3

샘플 데이터로 미리보기

Apps Script 편집기에서 insertSampleData 함수를 선택 후 ▶ 실행하면 데모용 데이터가 3개 시트에 채워집니다. 실제 방문자 데이터가 쌓이기 전에 대시보드 화면을 미리 확인할 때 유용해요.

4

코드를 고치면 반드시 "새 배포"

tracker.gs 코드를 수정한 뒤에는 배포 → 배포 관리 → 편집(연필) → 버전: 새 버전 → 배포 순으로 다시 배포해야 실제 웹앱에 반영됩니다. URL은 보통 그대로 유지돼요.
